    CPU: FDX Cycle

    When you run a program, the CPU loads or copies the program into random access memory (RAM). To execute the program, the central unit (CU) of the CPU fetches, decodes, and executes each instruction. The CPU uses high-speed memory locations called registers. These registers store the current instruction and associated data as it repeatedly iterates through this cycle for every instruction in the program. This instruction cycle, illustrated in this graphic, is referred to as the FDX (Fetch, Decode, Execute) cycle.
